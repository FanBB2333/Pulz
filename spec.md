# Pulz

**Pulz** 是一个借助中医诊断哲学（望、闻、问、切），辅助 LLM / Coding Agent 分析、理解并优化代码的智能工具。  
目标是提升代码质量、可维护性与系统健壮性，同时降低人工 Debug 时间。

---

## 1. 核心理念

| 中医诊断手段                   | 技术映射 / Coding 隐喻         | 功能示例                                                  |
| ------------------------------ | ------------------------------ | --------------------------------------------------------- |
| **望（Observation）**          | 静态代码分析、代码结构可视化   | 分析函数长度、循环嵌套深度、依赖关系，发现潜在复杂点      |
| **闻（Listening / Smelling）** | 运行时监控、日志分析           | 捕捉异常日志、性能瓶颈、异常模式                          |
| **问（Inquiry）**              | 交互式代码询问、上下文提问     | 自动生成注释、推断函数用途、识别潜在意图偏差              |
| **切（Pulse / Palpation）**    | 动态调试、数据流追踪、单元测试 | 检查变量状态、调用链、边界条件，发现隐藏 Bug 或潜在崩溃点 |

---

## 2. 功能模块

### 2.1 代码健康诊断 (Pulse Analysis)
- 静态分析：
  - 函数/类复杂度
  - 依赖耦合度
  - 潜在循环引用 / 死锁点
- 动态分析：
  - 日志异常聚类
  - 运行时资源消耗趋势
  - 单元测试覆盖率与异常模式

### 2.2 修复与优化建议 (Treatment)
- 自动生成 Patch / Refactor 建议
- 指导式重构路径推荐（最小修改量 / “扎针式修复”）
- 性能优化建议（缓存、异步、内存泄漏检测）

### 2.3 可视化与交互
- 系统脉搏图（Pulse Graph）展示代码健康状态
- 可视化调用链、依赖图
- 动态提示与交互式 Q&A，类似中医诊断问答

### 2.4 学习与自适应
- 基于历史代码库和修复记录，持续学习最有效的修复策略
- 对不同开发者的编码风格进行适应，提出个性化优化方案

## 3. 技术规格

### 3.1 输入
- 项目源码 / 文件路径
- 可选日志文件、单元测试报告
- 可选上下文信息（框架、语言版本、依赖库）

### 3.2 输出
- 健康评分 / Pulse Score
- Bug 风险点列表
- 修复 / 重构建议
- 可视化报告（Pulse Graph）

### 3.3 内核方法
- **LLM 辅助分析**：代码理解、上下文推理、生成可执行修复建议
- **静态分析**：AST 分析、复杂度计算、依赖图生成
- **动态分析**：日志聚合分析、运行时事件追踪、单元测试反馈分析
- **策略优化**：基于历史数据、代码风格、项目架构生成最优修复策略

### 3.4 平台 / 接口
- VSCode 插件 / Chrome 插件 / CLI 工具
- API 接口支持 LLM Agent 对接（Copilot, CodeBuddy, ChatGPT 等）
